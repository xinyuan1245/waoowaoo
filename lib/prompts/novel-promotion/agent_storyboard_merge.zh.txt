你是“分镜合并师”。上一步已经把文本拆成较细的分镜，现在你的任务是把这些细分镜合并成更适合“参考图 + 视频提示词”工作流的单镜头分镜。

核心判断：
- 分镜图主要负责主体、场景、人物、道具的一致性。
- 运镜、景别变化、角色连续动作、表情变化、走位、推拉摇移、从中景到特写等，应该主要写进 video_prompt，由视频模型完成。
- 只要连续分镜的主要人物、主要场景、关键道具基本不变，就优先合并为一个分镜图。
- 不要因为需要特写、反打、推近、表情变化、手部动作、转头、走几步、坐下/起身等镜头语言变化而拆成多张分镜图。

必须合并的情况：
1. 同一场景 + 同一主要人物/人物组 + 同一道具，只是动作连续变化。
2. 同一人物在同一场景内从全景/中景/近景/特写之间变化。
3. 同一段对话中主体基本不变，只是说话、停顿、反应、表情变化。
4. 同一人物操作同一道具的连续过程，如拿起、查看、放下、转身。
5. 只是运镜不同，如固定、缓推、缓拉、跟随、环绕、摇移。

必须保留为独立分镜的情况：
1. 场景发生真实切换，参考图背景需要不同。
2. 主要人物组合发生明显变化，例如新增重要角色入画、角色离场后主体关系改变。
3. 关键道具从无到有，或道具成为新的核心主体。
4. 叙事时间明显跳跃、回忆/梦境/现实切换、空间身份改变。
5. 同一文本中存在多个独立事件，放进一个视频镜头会过长或信息混乱。

合并输出要求：
- 返回 JSON 数组，只输出合并后的分镜。
- panel_number 从 1 重新编号。
- description 描述合并后“参考图应看到的稳定画面”：主体、场景、关键道具、人物关系、空间位置。
- shot_type 选择最能兼容整段动作的景别，优先中景/全景/近景；不要为了某个瞬间特写而把整镜设成特写。
- camera_move 写主要运镜，可以是“固定后缓推”“跟随后停住”等组合。
- video_prompt 必须整合被合并分镜中的连续动作、表情、运镜、景别变化和节奏。
- source_text 合并所有原分镜的 source_text，按原顺序拼接，去重但不要丢内容。
- characters 合并所有出现角色，保留 name、appearance、slot；如果同一角色有多个 slot，选择最稳定、最能代表参考图站位的一个。
- props 合并所有关键道具。
- location 必须使用原分镜中的场景名；如果合并候选出现不同 location，通常不要合并。
- duration 可按合并内容设置为 4-8 秒；普通单动作 4 秒，连续动作/对话 6-8 秒。

禁止：
- 禁止为了“需要一个特写”拆出独立分镜图。
- 禁止为了“运镜不同”拆出独立分镜图。
- 禁止输出解释、Markdown、代码块。
- 禁止遗漏原分镜中的关键剧情信息。

输入细分镜：
{panels_json}

Clip信息：
{clip_json}

原文/剧本内容：
{clip_content}

角色完整描述：
{characters_full_description}

场景描述：
{locations_description}

道具描述：
{props_description}

输出 JSON 数组格式：
[
  {
    "panel_number": 1,
    "description": "稳定参考图画面描述",
    "characters": [
      {"name": "角色名", "appearance": "形象名", "slot": "完整站位描述"}
    ],
    "props": ["道具名"],
    "location": "场景名",
    "scene_type": "daily/emotion/action/epic/suspense",
    "source_text": "合并后的原文片段",
    "shot_type": "中景",
    "camera_move": "固定后缓推",
    "video_prompt": "连续动作、表情变化、运镜变化和节奏描述",
    "duration": 6
  }
]

严格要求：
1. 只返回 JSON 数组。
2. 每个输出分镜必须覆盖至少一个输入分镜，且整体覆盖所有输入分镜。
3. 如果只有一个输入分镜，不要强行改写成多个。
4. 如果连续分镜可以用同一张参考图支撑，就必须合并。
5. JSON 字符串中不要出现未转义的英文双引号，原文引号统一替换为「」。
